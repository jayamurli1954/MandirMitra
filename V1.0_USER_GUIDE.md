## Getting Started

### First-Time Login

1. **Access the System**
   - Open your web browser
   - Navigate to: `http://localhost:3000` (or your server URL)
   - You'll see the login page

2. **Default Credentials**
   - Contact your system administrator for initial login credentials
   - Default admin credentials are typically:
     - Email: `admin@temple.com`
     - Password: (provided by administrator)

3. **Login Process**
   - Enter your email address
   - Enter your password
   - Click "Login"
   - You'll be redirected to the Dashboard

4. **Change Password (First Time)**
   - After first login, you may be prompted to change your password
   - Enter your current password
   - Enter new password (minimum 12 characters, with uppercase, lowercase, number, special character)
   - Confirm new password
   - Click "Update Password"

### Understanding the Interface

#### Main Navigation
- **Dashboard** - Overview of key metrics
- **Donations** - Donation management
- **Sevas** - Seva booking and management
- **Devotees** - Devotee CRM
- **Accounting** - Financial management
- **Inventory** - Stock management
- **Assets** - Asset management
- **HR** - Employee management
- **Reports** - All reports
- **Settings** - System configuration

#### Top Bar
- **User Profile** - Click to view profile, change password, logout
- **Notifications** - System notifications
- **Search** - Quick search across modules

## User Management

### Adding a New User

**Access:** Settings ‚Üí Users ‚Üí Add User

1. Click "Add User" button
2. Fill in user details:
   - **Full Name** (required)
   - **Email** (required, unique)
   - **Phone** (optional)
   - **Role** (select from dropdown):
     - Super Admin
     - Temple Manager
     - Accountant
     - Counter Staff
     - Priest
   - **Password** (temporary password)
3. Click "Create User"
4. User will receive credentials (if email configured)

### Editing User Details

1. Go to Settings ‚Üí Users
2. Find the user in the list
3. Click "Edit" icon
4. Modify details
5. Click "Save"

### Deactivating a User

1. Go to Settings ‚Üí Users
2. Find the user
3. Click "Deactivate"
4. Confirm deactivation
5. User will no longer be able to login

### Changing Your Password

1. Click your profile icon (top right)
2. Select "Change Password"
3. Enter current password
4. Enter new password
5. Confirm new password
6. Click "Update"

## Donation Management

### Recording a New Donation

**Access:** Donations ‚Üí New Donation

#### Step-by-Step Process:

1. **Select Devotee**
   - Type devotee name or phone number
   - System will auto-suggest matching devotees
   - Select from dropdown
   - OR click "Add New Devotee" to create new entry

2. **Enter Donation Details**
   - **Donation Category** (required)
     - Select from dropdown (General Donation, Annadanam, Construction Fund, etc.)
   - **Amount** (required)
     - Enter donation amount
   - **Payment Mode** (required)
     - Cash
     - Card
     - UPI
     - Cheque
     - Online
   - **Donation Date** (defaults to today)
     - Select date from calendar
   - **Transaction ID** (if online/UPI)
     - Enter transaction reference
   - **Cheque Details** (if cheque payment)
     - Cheque Number
     - Cheque Date
     - Bank Name
   - **Notes** (optional)
     - Any additional information

3. **80G Certificate**
   - Check "Eligible for 80G" if applicable
   - System will track for certificate generation

4. **Save Donation**
   - Click "Save Donation"
   - Receipt will be automatically generated
   - Receipt number will be displayed

### Viewing Donations

**Access:** Donations ‚Üí Donation List

**Features:**
- Filter by date range
- Filter by category
- Filter by payment mode
- Filter by devotee
- Search by receipt number
- Sort by date, amount, etc.

**Actions Available:**
- View receipt
- Edit donation (if not locked)
- Cancel donation
- Print receipt

### Generating Receipts

1. Go to Donations ‚Üí Donation List
2. Find the donation
3. Click "View Receipt"
4. Receipt will open in new tab (PDF)
5. Click "Print" to print
6. Click "Download" to save PDF

### Bulk Import of Donations

**Access:** Donations ‚Üí Import

1. **Prepare CSV File**
   - Download template
   - Fill in columns:
     - Devotee Phone/Email
     - Category
     - Amount
     - Payment Mode
     - Date
     - Transaction ID (if applicable)

2. **Upload File**
   - Click "Choose File"
   - Select your CSV file
   - Click "Upload"

3. **Review Import**
   - System will show preview
   - Check for errors
   - Fix any issues

4. **Confirm Import**
   - Click "Import"
   - System will process all donations
   - Receipts will be auto-generated

### 80G Certificate Generation

**Access:** Donations ‚Üí 80G Certificates

1. **Select Financial Year**
   - Choose financial year from dropdown

2. **Select Devotee**
   - Choose devotee or select "All Devotees"

3. **Generate Certificates**
   - Click "Generate Certificates"
   - System will create PDF certificates
   - Download individual or batch

4. **Send Certificates**
   - Select certificates
   - Click "Send via Email" (if configured)
   - OR download and send manually

### Cancelling a Donation

1. Go to Donations ‚Üí Donation List
2. Find the donation
3. Click "Cancel"
4. Enter cancellation reason
5. Confirm cancellation
6. Receipt will be marked as cancelled

## Accounting System

### Chart of Accounts

**Access:** Accounting ‚Üí Chart of Accounts

#### Understanding Account Types:

- **Assets** - What temple owns
- **Liabilities** - What temple owes
- **Income** - Money received
- **Expenses** - Money spent
- **Capital** - Temple's capital/funds

#### Adding a New Account:

1. Click "Add Account"
2. Fill in details:
   - **Account Name** (required)
   - **Account Code** (auto-generated or manual)
   - **Account Type** (select from dropdown)
   - **Parent Account** (for sub-accounts)
   - **Description** (optional)
3. Click "Save"

#### Account Linking:

Accounts can be linked to:
- Donation Categories
- Seva Types
- Expense Categories
- Income Categories

**To Link:**
1. Go to Settings ‚Üí Account Linking
2. Select category
3. Select account
4. Click "Link"

### Journal Entries

**Access:** Accounting ‚Üí Journal Entries

#### Types of Journal Entries:

1. **Receipt Entry** - Money received
2. **Payment Entry** - Money paid
3. **Journal Entry** - Transfer between accounts
4. **Contra Entry** - Cash/Bank transfers

#### Creating a Receipt Entry:

1. Click "New Entry" ‚Üí "Receipt"
2. Fill in details:
   - **Date** (required)
   - **Voucher Number** (auto-generated)
   - **Received From** (account/person)
   - **Amount** (required)
   - **Payment Mode** (Cash/Bank)
   - **Narration** (description)
3. **Debit Account** (auto-selected based on payment mode)
4. **Credit Account** (select income account)
5. Click "Save"

#### Creating a Payment Entry:

1. Click "New Entry" ‚Üí "Payment"
2. Fill in details:
   - **Date** (required)
   - **Paid To** (account/person)
   - **Amount** (required)
   - **Payment Mode** (Cash/Bank)
   - **Narration** (description)
3. **Debit Account** (select expense account)
4. **Credit Account** (auto-selected based on payment mode)
5. Click "Save"

#### Creating a Journal Entry:

1. Click "New Entry" ‚Üí "Journal"
2. Fill in details:
   - **Date** (required)
   - **Narration** (description)
3. **Add Debit Lines:**
   - Select account
   - Enter amount
   - Click "Add"
4. **Add Credit Lines:**
   - Select account
   - Enter amount
   - Click "Add"
5. **Verify:** Total Debit = Total Credit
6. Click "Save"

### Automatic Transaction Posting

The system automatically creates journal entries for:
- Donations ‚Üí Income account
- Seva Bookings ‚Üí Income account
- Expenses ‚Üí Expense account
- Payments ‚Üí Bank/Cash account

These are created automatically when transactions are recorded.

### Viewing Journal Entries

1. Go to Accounting ‚Üí Journal Entries
2. Filter by:
   - Date range
   - Entry type
   - Account
   - Voucher number
3. View details
4. Edit (if not locked)
5. Delete (if not locked)

## Hundi Management

   - **Scheduled Date** (required)
   - **Scheduled Time** (required)
   - **Location** (optional)

### Opening a Hundi

1. Go to "Scheduled" tab
2. Find the hundi
3. Click "Open"
4. System will record:
   - Opening time
   - Opening person
   - Sealed number (if applicable)

### Counting Hundi

1. After opening, click "Start Counting"
2. **Enter Denomination Counts:**
   - ‚Çπ2000 notes
   - ‚Çπ500 notes
   - ‚Çπ200 notes
   - ‚Çπ100 notes
   - ‚Çπ50 notes
   - ‚Çπ20 notes
   - ‚Çπ10 notes
   - Coins (‚Çπ10, ‚Çπ5, ‚Çπ2, ‚Çπ1)

4. Click "Save Count"

### Multi-Person Verification

1. After first count, click "Verify"
2. **Second Person Verification:**
   - Enter second person name
   - Re-count denominations
   - Click "Verify"
3. **Third Person Verification (if required):**
   - Enter third person name
   - Re-count denominations
   - Click "Verify"

5. If discrepancies found, system will highlight

### Handling Discrepancies

1. If counts don't match:
   - System will show difference
   - Re-count required
   - Enter discrepancy reason
2. After re-count:
   - Update counts
   - System will verify
3. Once verified, proceed to deposit

### Recording Bank Deposit

1. After verification, click "Record Deposit"
2. Fill in details:
   - **Bank Account** (select)
   - **Deposit Date** (required)
   - **Deposit Amount** (auto-filled)
   - **Transaction Reference** (optional)
3. Click "Save Deposit"
4. System will create accounting entry

### Reconciliation

1. Go to "Deposits" tab
2. Find the deposit
3. Click "Reconcile"
4. Match with bank statement
5. Mark as reconciled
6. System will update status

### Viewing Hundi Reports

1. Go to "Reports" tab
2. View:
   - Hundi opening schedule
   - Counting summary
   - Deposit summary
   - Reconciliation status
3. Export reports

## Financial Closing

**Access:** Accounting ‚Üí Financial Closing

### Month-End Closing

1. **Review Transactions:**
   - Ensure all transactions entered
   - Verify all receipts/payments recorded

2. **Run Reports:**
   - Trial Balance
   - Profit & Loss
   - Balance Sheet
   - Verify balances

3. **Close Month:**
   - Go to Financial Closing
   - Select month
   - Click "Close Month"
   - Confirm closing
   - System will:
     - Lock period
     - Create closing entries
     - Update financial year status

### Year-End Closing

1. **Complete All Months:**
   - Ensure all months closed
   - Review year-end reports

2. **Year-End Procedures:**
   - Finalize all transactions
   - Complete bank reconciliation
   - Verify all accounts

3. **Close Financial Year:**
   - Go to Financial Closing
   - Select financial year
   - Click "Close Year"
   - Confirm closing
   - System will:
     - Lock entire year
     - Create year-end entries
     - Carry forward balances

### Opening New Financial Year

1. **Create New Financial Year:**
   - Go to Financial Closing
   - Click "New Financial Year"
   - Enter:
     - Year (e.g., 2025-26)
     - Start Date
     - End Date

2. **Set Opening Balances:**
   - Go to Chart of Accounts
   - For each account, set opening balance
   - System will carry forward from previous year

3. **Verify Opening Balances:**
   - Run Trial Balance
   - Verify all balances correct
   - Make adjustments if needed

### Period Locking

**Q: How do I backup my data?**

A: No, v1.0 requires internet connection. Offline mode is planned for v2.0.

A: No limit on number of users. System supports multiple concurrent users.

A: Standard reports are available. Custom reports can be requested for v2.0.
