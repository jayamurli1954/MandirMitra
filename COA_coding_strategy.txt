## ‚úÖ What is the **GENERAL PRACTICE** for COA Coding?

### Short Answer

There is **NO single universal standard**.

**General practice worldwide follows only 2 fixed rules:**

1. **Accounts are grouped by nature (Asset, Liability, etc.)**
2. **Codes are mainly for machines, not humans**

Everything else varies.

---

## üåç General Practice by Segment

### 1Ô∏è‚É£ Traditional Manual Accounting (Old Era)

* **No codes at all**
* Account names only
* Ledgers arranged physically

‚û°Ô∏è Still followed by **very small firms**

---

### 2Ô∏è‚É£ Tally (Most common in India üáÆüá≥)

#### Practice:

* **No mandatory COA codes**
* Ledger **name is primary**
* Grouping decides report behavior

Example:

```
Ledger Name: Cash
Group: Cash-in-Hand
```

‚úî User friendly
‚ùå Difficult for APIs, analytics, migration

üëâ **Reality**: 70‚Äì80% Indian businesses do NOT use codes

---

### 3Ô∏è‚É£ ERP / Professional Accounting Software (SAP, Oracle, NetSuite)

#### Practice:

* **Numeric COA codes are mandatory**
* Usually **4 to 6 digits**
* First digit = Account class

Typical SAP-like structure:

```
100000 ‚Äì Assets
200000 ‚Äì Liabilities
300000 ‚Äì Equity
400000 ‚Äì Revenue
500000 ‚Äì Expenses
```

Sub-ledgers extend underneath.

‚úî Machine-friendly
‚úî Scalable
‚ùå Not beginner friendly

---

### 4Ô∏è‚É£ International Best Practice (IFRS / US GAAP)

**Very common structure:**

| Class       | Range     |
| ----------- | --------- |
| Assets      | 1000‚Äì1999 |
| Liabilities | 2000‚Äì2999 |
| Equity      | 3000‚Äì3999 |
| Income      | 4000‚Äì4999 |
| Expenses    | 5000‚Äì5999 |

Used by:

* Auditors

* Large enterprises

üëâ **This is the closest thing to ‚Äúgeneral practice‚Äù**

## üáÆüá≥ Indian Mid-Market Reality (CA Firms)
